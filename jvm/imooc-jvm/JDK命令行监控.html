# 基于JDK命令行的监控

# JVM 的参数类型

# 标准参数

jvm 的基础参数, 几乎不变

例如:

# 查看 java 的版本信息
java -version 

# 查看帮助信息
java -help

# X 参数

非标准化参数, 各 jvm 的版本中可能会会变, 但变化的可能比较小

例如:

# java 执行方式--完全解释执行
-Xint

# java 执行方式--第一次就编译成本地代码
-Xcomp

# java 执行方式--混合模式, JVM 自己决定是否编译成本地代码(默认)
-Xmixed

# XX 参数

非标准化参数, 相对不稳定, 主要用于 JVM 调优和 Debug

  • Boolean 类型

    格式:

    # 启用/禁用 name 属性
-XX:[+-]<name>
    1
    2

    例如:

    # 启用 CMS 垃圾回收器
-XX:+UseConcMarkSweepGC

# 启用 G1 垃圾回收器
-XX:+UseG1GC
    1
    2
    3
    4
    5

  • 非 Boolean 类型

    语法:

    # 表示 name 属性的值是 value
-XX:<name>=<value>
    1
    2

    例如:

    # 是允许的GC最大的暂停时间。
# 首先，我们能容忍的最大暂停时间是有一个限度的，我们需要在这个限度范围内设置。但是应该设置的值是多少呢？我们需要在吞吐量跟MaxGCPauseMillis之间做一个平衡。如果MaxGCPauseMillis设置的过小，那么GC就会频繁，吞吐量就会下降。如果MaxGCPauseMillis设置的过大，应用程序暂停时间就会变长。G1的默认暂停时间是200毫秒，我们可以从这里入手，调整合适的时间。
-XX:MaxGCPauseMillis=500

# 设置吞吐量大小，它的值是一个 0-100 之间的整数。假设 GCTimeRatio 的值为 n，那么系统将花费不超过 1/(1+n) 的时间用于垃圾收集。
# 例如, 当前设置的值是 19, 那系统将不花费超过 1/(1+19) 也就是 20% 的资源用于垃圾收集
-XX:GCTimeRatio=19

# 初始化的堆大小, 这是一个 XX 参数, 等价于 -XX:InitialHeapSize
-Xms

# 最大的堆的大小, 这是一个 XX 参数, 等价于 -XX:MaxHeapSize
-Xmx

# 线程堆栈的大小, 这是一个 XX 参数, 等价于 -XX:ThreadStackSize
-Xss

# jps

虚拟机进程状态查看工具(可以远程查看)

语法

jps [options] [hostid]
1

查看当前正在运行的 java 进程

例如

# 查看简要信息
jps

# 查看列表详细信息
jps -l

# jinfo

可以用来查看正在运行的Java应用程序的扩展参数，甚至支持在运行时，修改部分参数

语法:

# 基础语法
jinfo [option] pid
1
2

命令:

# 查看参数(单个)
jinfo -flag name
# 查看所有修改后的参数
jinfo -flags

# 修改参数(Boolean 类型)
jinfo -flag [+-]name
# 修改参数(非 Boolean 类型)
jinfo -flag name=value

# 打印 java 的属性
jinfo -sysprops
1

# jstat

查看 JVM 的类装载信息, 垃圾收集信息, JIT 编译信息

语法:

# 分为单参数和多参数
jstat [ generalOption | outputOptions vmid [ interval [s | ms] [ count ]]
1
2

单参数语法:

jstat -help | -options
1

命令

# 显示有关类加载器行为的统计信息
jstat -class

# 显示有关Java HotSpot VM实时编译器行为的统计信息。
jstat -compiler

# 显示有关垃圾回收堆行为的统计信息。
jstat -gc

# 显示Java HotSpot VM编译方法统计信息。
jstat -printcompilation
1

# 显示 pid 为 1549 的进程的类加载的信息, 每隔 1000sm 输出一次, 一共输出 10 次
jstat -class 1549 1000 10
