# 技术规划&工作中的一些思考

加入"跬步信息"已两月有余, 在这里, 我见到了一个之前没有想到过的行业, 也学习了不少的不一样(过去不关注或了解比较少)的技术, 一直想写一下在公司的见闻, 以及由这些见闻所引发的思考. 正值年终之际, 隧把这些一起写入自己的总结.

# 技术规划

# 语言本身

关键词: Java基础 Jvm

刚开始学习 java 的时候, 授课的老师讲的第一条就是 学习计算机语言, 都是由多到少, 再少到多, 然后再由多到少的过程 , 听起来很绕哈. 其实就是说, 开始学习时觉得需要学习的内容很多. 随着学习的推进, 掌握了冰山一角的学生会觉得"也不过如此". 但随着学习的进一步推进才发现"原来还有整座冰山藏在水中". 但经过漫长的学习之后, 已经看遍整座山的人会说: "确实不过如此"

引入上面 "第一课" 的内容, 是想说明一下自己所处的位置, 我还在 "探索冰山" 的过程中. 所以在未来的一年, 我会对 java语言 本身再做更深一步的学习. 主要的学习方向是 java 的设计思想和设计模式. 设计模式本身的学习并不复杂, 关键的问题是, 根据不同的场景选择对应的优秀的设计模式. 这方面的学习方式, 我打算结合优秀的开源项目进行实战性质的学习.

之前对 Jvm 的学习, 是停留在, 了解主要逻辑, 了解主要解析过程的阶段, 下一步就是对 Jvm 的解析过程的细节, 做更深一步的了解. 针对 Jvm 的学习, 我计划由 1 ~ 2 本比较系统的书籍作为基础, 再结合博客上的内容来学习.

# 架构

关键词: SpringBoot SpringCloud Dubbo

回想起自己用过的框架, 从 Spring + Struts2 + Hibernate 到 Spring + SpringMVC + MyBatis 再到 Spirng + SpringMVC + Hibernate 以及 Spring + SpringMVC + Hibernate + DBHelp , 用过的框架也不算少了, 但自从看到了 SpirngBoot 感觉以前的好多做法都是在浪费生命. 所以毫不犹豫的投入了 SpringBoot 的怀抱.

再好的框架也是需要学习的. 所以, 接下来框架的学习重点将会是 SpringBoot , 学习的方式是, 以 1 ~ 2 本系统学习的书籍, 结合官网文档(提到官方文档, 我忍不住要赞扬一下, Spring 的官方文档, 无论从可读性还是逻辑性都是非常优秀的), 再结合博客上的内容进行学习.

# 代码管理

关键词: git

说来惭愧, git 这么好的工具, 我并没有了解的特别深入, 因为之前的项目管理都是直接提交, 分支的维护需求几乎没有, 所以也没有过多的关注这些内容. 结果在 "公有云" 项目切换的时候, 深深的坑了自己一次.

本着同样的错误不能犯两次的原则, 我也要系统的学习一下 git .

PS: 这块技术的学习已经在进行了.

# 代码规范

关键词: 阿里代码规范

规范的代码是团队之间互相信赖的基础. 同时,规范的代码也能很大程度上提升团队的工作效率. 所以规范代码是特别有必要甚至是必须的事情.

在 java 代码规范方面, 阿里的代码规范并不一定是最好最完善, 但代码规范本身就是一种约定, 只要大家都遵守相同的约定, 就能让团队之间互相信任, 就能提升团队的效率.

在代码规范方面, 我需要学习的是更多的细节, 还有对代码规范制定的深意的理解上.

# 基础

关键词: HTTP 语言结构及发展

欠下到账, 早晚都是要还的. 对一个非本专业的人来说, 基础知识的薄弱是一个特别常见的现象, 虽然在开发过程中也零零散散的学过一些, 但终不成系统. 所以在未来的学习中, 也需要把这些"曾经欠下的帐"换掉, 需要系统的学习一下 HTTP 请求的原理, 过程以及过程中的一些细节内容.

另外, 我还对语言发展的本身有很大的兴趣, 这方面的学习主要是通过语言发展的历程, 了解新的语言和上一代语言在设计上的差异, 以及是什么原因造成了这样的差异, 这样以后, 我们得到了什么, 又失去了什么.

这就是我在未来一年的技术规划.

# 一些思考

# 最长的适应期

人都说, 换一个新的环境, 总会有一段适应期. 但是在跬步, 我经历了工作以来最长的适应期, 这种适应并不是和公司的同事之间, 而是在业务和技术上. 为什么会出现这样的情况? 接下来我将从如下几个方面来详细的说明.

# 从吐槽到沉默

记得刚开始接手 "私有云" 项目的时候, 通过对项目的简要了解, 我觉得这是一个不复杂的项目, 模块不多, 数据并发量也不算大, 做起来应该是蛮轻松的.

但通过进一步的查看代码, 进一步对项目了解, 我发现我错了. 项目中各种不按常理出牌, 各种不符合规范的处理方式, 让我觉得异常的痛苦. 相信不止是我, 这样的项目也会让其他新加入的成员感到痛苦.

于是我开始吐槽.
